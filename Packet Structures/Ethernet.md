# Ethernet II Frame Structure

```text
####################################################################################################
                                     Ethernet II Frame 
####################################################################################################

┌────────────┬──────────┬──────────────────────────┬──────────────────────────┬──────────────────────┬──────────────────────────────────────────┬───────────┐
│  Preamble  │   SFD    │     Destination MAC      │        Source MAC        │      EtherType       │           Payload + Padding              │    FCS    │
└────────────┴──────────┴──────────────────────────┴──────────────────────────┴──────────────────────┴──────────────────────────────────────────┴───────────┘

Preamble (7 bytes)
 : 송수신 장치가 비트 타이밍을 맞추기 위한 동기 패턴
   예시 → AA AA AA AA AA AA AA

SFD – Start Frame Delimiter (1 byte)
 : 프레임 시작 표시
   예시 → AB

Destination MAC (6 bytes)
 : 목적지 MAC 주소
   예시 → ff:ff:ff:ff:ff:ff

Source MAC (6 bytes)
 : 출발지 MAC 주소
   예시 → 08:00:27:12:34:56

EtherType (2 bytes)
 : 상위 프로토콜 식별 필드
   예시 → 0x0800 (IPv4)
           0x0806 (ARP)
           0x86DD (IPv6)

Payload + Padding (46 ~ 1500 bytes)
 : 실제 전송 데이터 + 최소 프레임 길이(64 bytes) 충족을 위한 패딩
   예시 → [IPv4 Header + TCP Data] + 00 00 ...

FCS – Frame Check Sequence (4 bytes)
 : CRC32 기반 오류 검출 값
   예시 → 3A 7B 19 FE
