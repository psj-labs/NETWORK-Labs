# Network Basic & TCP/IP 이해

- **인터넷은 전 세계 수백만 대의 컴퓨팅 장치(Host)를 연결한 '네트워크의 네트워크'이며, 프로토콜을 통해 데이터의 송수신을 제어하는 복합적인 통신 기반 구조입니다.**

---

## 네트워크란?

- 네트워크는 서로 다른 시스템(호스트)에서 실행 중인 프로세스들 간에 메시지를 주고받게 해줍니다.
- 우리가 사용하는 네트워크는 `클라이언트 : 서버` 통신입니다.
- Ex) A가 B에게 카카오톡 메시지를 보낼때 실제로는 A의 카카오톡 클라이언트가 카카오톡 서버로 메시지를 보내고, 서버가 이를 B의 클라이언트로 전달하는 것입니다.

---

## 종단시스템이란?

- 네트워크의 가장 끝에 위치해 실제로 통신을 수행하는 장치이며 프로세스가 실행되는 시스템을 말합니다.

---


## 인터넷의 구성 요소와 프로토콜

- 인터넷은 종단 시스템(Host), 패킷 스위치(Router, Switch), 그리고 통신 링크로 구성됩니다.
- **프로토콜(Protocol):** 둘 이상의 통신 개체 간에 교환되는 메시지 포맷, 순서, 그리고 메시지 송수신 시 취하는 행동을 정의한 약속입니다. (예: TCP, IP, HTTP 등)


---

## 네트워크 구조: Edge와 Core

- 네트워크는 역할에 따라 가장자리(Edge)와 중심부(Core)로 구분됩니다.

### 1. Network Edge
- 호스트(Host)와 응용 프로그램(Web, Email 등)이 실행되는 영역입니다.
- **Client/Server 모델:** 클라이언트가 요청하고 서버가 응답하는 방식입니다.
- **P2P(Peer-to-Peer) 모델:** 클라이언트와 서버의 기능을 동시에 수행합니다.

### 2. Network Core
- 종단 시스템들을 연결하는 라우터들의 망입니다. 데이터 전송 방식에 따라 두 가지로 나뉩니다.
- **회선 교환(Circuit Switching):** 통신 세션 동안 전용 회선을 예약하여 성능을 보장하지만 자원 낭비가 발생할 수 있습니다. (FDM, TDM 방식 이용)
- **패킷 교환(Packet Switching):** 데이터를 패킷 단위로 나누어 전송하며 자원을 공유합니다. 자원 효율성이 높지만 혼잡 시 지연이나 손실이 발생할 수 있습니다.

---

## 패킷 교환의 특성과 전달 방식

- 패킷 교환은 통계적 다중화를 통해 불규칙한 데이터 전송에 효율적으로 대응합니다.

- **데이터 전달 방식:**
    - **Datagram Network:** 목적지 주소만을 이용하여 독립적으로 경로를 결정합니다. (비연결형)
    - **Virtual Circuit(VC):** 출발지부터 목적지까지 미리 경로를 확립하고 식별자(ID)를 이용해 전달합니다. (연결지향형)


## Statistical Multiplexing (통계적 다중화)


- 통계적 다중화란 여러 호스트가 동시에 최대 대역폭을 사용하지 않는다는 확률적 특성을 이용해 하나의 전송 링크를 동적으로 공유하는 방식입니다.

---

## 패킷 교환 : 통계적 다중화
- 통계적 다중화는 **패킷 교환(Packet Switching)** 네트워크의 핵심 개념입니다.
- 각 호스트는 고정된 시간 슬롯을 할당받지 않으며 **패킷이 도착하는 순서대로 큐(queue)에 저장된 후 출력 링크로 전송**됩니다.

즉,
- 누가 먼저 보낼지
- 어떤 호스트의 패킷이 연속으로 나갈지  
-> 사전에 정해진 규칙이 없습니다.

#### 이 불규칙성이 바로 “통계적(statistical)”이라는 이름의 이유입니다.

---

## 그림 기반 동작 해설

![statistical multiplexing](/Network/imgs/statistical%20multiplexing.png)

- 호스트 A, B는 각각 **10 Mb/s Ethernet**으로 패킷을 생성
- 이 패킷들은 라우터 내부의 **출력 큐(queue)**에 저장됨
- 출력 링크는 **1.5 Mb/s**로 제한되어 있음
- 패킷은 **도착한 순서대로** 출력 링크를 통해 전송됨

### 핵심 포인트
- A와 B의 패킷 전송 순서는 **일정하지 않습니다.**
- 반복되는 패턴이나 고정 슬롯이 존재하지 않습니다
- 트래픽이 몰리면 큐에 패킷이 쌓이고, 지연이 발생할 수 있습니다

---

## TDM(Time Division Multiplexing)과 비교

| 구분 | TDM | 통계적 다중화 |
|---|---|---|
| 슬롯 할당 | 고정 | 없음 |
| 전송 순서 | 반복적 | 비결정적 |
| 유휴 자원 | 자주 발생 | 거의 없음 |
| 효율성 | 낮음 | 높음 |
| 사용 예 | 전화망 | 인터넷(IP 네트워크) |

### TDM 특징
- 각 호스트는 반복되는 프레임 안의 **고정 슬롯**을 할당받음
- 전송할 데이터가 없어도 슬롯은 비워진 채 유지됨

### 통계적 다중화 특징
- 슬롯 개념이 없음
- 전송할 데이터가 있는 호스트만 링크 사용
- 링크 활용률이 매우 높음

---

## 장점과 단점

### 장점
- 전송 링크 활용률 극대화
- 다양한 트래픽 패턴에 유연하게 대응 가능

### 단점
- 트래픽이 동시에 몰릴 경우 큐잉 지연 발생
- 버퍼가 가득 차면 패킷 손실 발생 가능

---

## 네트워크 접속 기술

- 종단 시스템을 가장자리 라우터(Edge Router)에 연결하는 방법입니다.

| 접속 유형 | 특징 |
|:---|:---|
| **가정 접속** | DSL(전화선 이용, 비대칭 전송), 케이블 모뎀(HFC 기술), FTTH(광케이블) |
| **기관 접속** | Ethernet(대학교, 기업 내 LAN 구축), 10M~Gbps 속도 제공 |
| **무선 접속** | 무선 LAN(WiFi - 802.11), 광대역 무선망(3G, 4G, 5G 이동통신) |

---

## 물리 매체

- 비트는 물리적 매체상에 전파되는 파형으로 전달됩니다.
- **유도 매체:** 견고한 매체를 따라 신호를 유도합니다. (TP, 동축케이블, 광케이블)
- **비유도 매체:** 대기 공간으로 신호를 전파합니다. (무선 LAN, 위성 라디오)



---

## 인터넷의 구조: Network of Networks

- 인터넷은 계층적인 구조를 가진 ISP들의 집합입니다.
- **Tier-1 ISP:** 국가 및 국제적 범위를 가진 백본망 운영자들입니다.(ex: Sprint, AT&T)
- **Tier-2 / Tier-3 ISP:** 상위 ISP의 서비스를 구매하여 지역적인 접속 서비스를 제공합니다.
- 패킷은 목적지에 도달하기 위해 수많은 ISP망과 IXP(Internet Exchange Point)를 통과합니다.
